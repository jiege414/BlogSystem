# 博客系统测试计划

## 1. 项目概述
本项目是一个基于Flask的个人博客系统，已完成用户认证和文章管理核心功能。

### 1.1 项目功能
- **用户认证模块**：用户注册、登录、登出
- **文章管理模块**：文章创建、查看详情、编辑、删除
- **权限控制**：文章编辑/删除仅作者可见可操作
- **安全特性**：CSRF保护、密码哈希、会话管理

### 1.2 技术栈
- 后端框架：Flask 2.3.3
- 数据库：SQLite（通过 Flask-SQLAlchemy）
- 用户认证：Flask-Login
- 表单处理：Flask-WTF + WTForms
- 前端框架：Bootstrap 5.3.3

## 2. 测试目标
确保核心功能（用户注册/登录、文章增删改查）稳定、安全、符合需求。

### 2.1 功能完整性
- 所有功能模块正常工作
- 用户流程顺畅无阻
- 数据正确保存和显示

### 2.2 安全性
- 权限控制有效
- CSRF攻击防护
- 密码安全存储
- 会话管理正确

### 2.3 用户体验
- 界面友好易用
- 错误提示清晰
- 操作反馈及时

## 3. 测试范围
- **功能测试**：用户注册、登录、退出；文章的创建、查看、编辑、删除
- **权限测试**：验证文章编辑/删除的权限控制（仅作者可见可操作）
- **安全测试**：CSRF保护、SQL注入防护、会话管理、密码安全
- **用户体验测试**：删除确认对话框、表单预填充、表单验证提示
- **界面测试**：页面布局、响应式设计、导航功能
- **边界测试**：表单输入边界值、空数据、异常情况处理

## 4. 测试策略
- 采用黑盒测试方法，根据需求设计测试用例
- 先执行手动功能测试，再补充自动化测试
- 对发现的问题进行记录、跟踪和回归测试
- 测试优先级：功能测试 > 安全测试 > 用户体验测试

## 5. 测试环境
- **操作系统**：Windows 10/11
- **浏览器**：Chrome（主要）、Edge（兼容性测试）
- **Flask开发服务器**：http://127.0.0.1:5000
- **数据库**：SQLite（instance/blog.db）
- **Python版本**：Python 3.x
- **虚拟环境**：venv

## 6. 测试工具
- 浏览器开发者工具（F12）：用于检查网络请求、控制台错误
- SQLite数据库查看工具：用于验证数据存储
- 手动测试：主要测试方法

## 7. 测试数据准备
### 7.1 用户测试数据
- 有效用户：正常格式的用户名、邮箱、密码
- 无效用户：重复用户名、无效邮箱、短密码等
- 边界值：最小/最大长度限制

### 7.2 文章测试数据
- 正常文章：有效标题和内容
- 边界值：空标题、超长标题、空内容等
- 特殊字符：包含HTML标签、特殊符号的内容

## 8. 测试进度跟踪
- 测试用例执行状态：待执行/执行中/已通过/已失败
- Bug跟踪：记录在 BUG_REPORTS.md
- 测试记录：记录在 MANUAL_TESTING.md

## 9. 已知风险与说明
- 安全风险：当前测试发现 CSRF 校验在“创建文章/删除文章”场景未生效（见 BUG-SEC-001、BUG-SEC-002），存在安全隐患，建议优先修复并回归验证。
- 体验风险：登录后 next 参数重定向未按预期返回原目标页面（见 BUG-UX-001），影响受保护页面访问闭环，建议修复。
- 备注：本测试计划中提到的安全特性为“期望具备/需验证”的能力，实际以测试结果与缺陷记录为准。
